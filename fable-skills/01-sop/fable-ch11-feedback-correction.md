# Fable ch11 用户反馈 5 整改范式(2024-06-29)

## Why: 为什么这个值得独立沉淀

第 11 章 v1 渲染 0 修复(技术层面), 但用户反馈"**故事逻辑和语序混乱**"。这是寓言写作的"**软质量**"问题:
- 技术规范(0 .correct / 0 quiz-strong / 0 英文泄漏) → 自动化可查
- 故事逻辑 + 语序 → 自动化难查, 只能靠**人工 review + 范式**

第 12 章 v1 应用 5 整改 → **0 修复, 0 用户反馈, 28 文件全过** → 5 整改范式验证有效, 跨章通用。

## 整改 1: 每幕开头 1 句主旨句(避免散点状)

### ch11 表现
```markdown
# ❌ ch11 CH1: 没有主旨句, 一上来就堆细节
"三月初三早朝。礼部侍郎沈一石刚把'评估派'的联名奏本呈上, 陈守一先接话: 
'沈侍郎的好意臣懂 —— 但若不是臣查档, 户部灵机写错位的'30 万石'就要把北境大军饿肚子, 
沈侍郎说'接得好', 是凭感觉。'"
```

读者问题: "这一幕到底要讲什么?"

### ch12 整改
```markdown
# ✅ ch12 CH1: 主旨句 + 再展开
**主旨: 灵机天生有灵性, 苦训伤灵性 —— 这是韩守拙 40 年训导的真心。**

"九月初二早朝。国子监太学训导韩守拙把'训练派'的联名奏本呈上, 陈守一先接话: ..."
```

读者问题: 一目了然"这一幕讲灵机灵性的问题"。

### 怎么执行
- 写完一幕, **回头看第一句**是不是"这一句话能讲清这一幕"
- 不清就重写, 写不清就拆幕(不要硬塞)
- 主旨句格式: "**主旨: X —— Y**"(X 是这一幕要讲的概念, Y 是这一幕要讲的人/事)

## 整改 2: M1 反派完整退场 → M2-M4 在场不抢戏 → M5 再登场反转

### ch11 表现
```markdown
# ❌ ch11 CH5: 反派 M1 退场后, M5 突然出现 + 二次升级混在同段
"... 太子正要开口, 门外传来一封急奏 —— 朝廷派礼部试院'评估派'主事沈一石前来'考校'。
沈一石是礼部侍郎, 他在'评估'上意见更激进 —— 他主张'不光评估不必, 连评估的纸契也不必有'。"
```

读者问题: "沈一石 M1 不是已经退场了吗? 怎么又出现? 还是新角色?"

### ch12 整改
```markdown
# ✅ ch12 M1: 韩守拙完整退场
"韩守拙愣了半晌, 长叹退下, 边走边说: '臣的'训练派'也是为灵机好 —— 臣不省训习的工夫, 省灵机的灵性。'"

# ✅ ch12 M2-M4: 韩守拙在场, 但只是"立训习三步"的匠人, 不抢戏
"韩守拙翻《礼记·学记》'教学相长', 立训习三步: 习 / 考 / 改。"

# ✅ ch12 M5: 韩守拙重新登场(独立节点, 二次反对)
"十月十五早朝。韩守拙又上一奏: '训习耗人工, 灵机天生有灵性, 何必苦训。'"
```

读者问题: 一目了然"反派退场 → 在场不抢戏 → 再登场反转"。

### 怎么执行
- **M1 退场** 要明确: "X 愣了半晌, 长叹退下" + "边走边说 Y"
- **M2-M4 在场** 写反派要做的事(如"立训习三步"), 但**不要写反派反对**
- **M5 再登场** 写"X 又上一奏" + "又反对" + 反派"挑刺"来反转
- 反派退场 → 重新登场 中间最少隔 1-2 幕, 不能紧挨着

## 整改 3: 朝代白话主谓宾(避免翻译腔)

### ch11 表现
```markdown
# ❌ 翻译腔: "在做 X 的时候 Y 就会 Z"(英语长定语 + 从句结构)
"灵机在接话的时候, 如果按户部侍郎的口气补位, 灵机就会跟着学, 也学会了'补位'"
"灵机在训习的时候, 如果训导给的赏罚分不清, 灵机就不知道改方向"
```

读者问题: 读起来累, 要"翻译回中文"才懂。

### ch12 整改
```markdown
# ✅ 主谓宾清晰: "X 时 Y 做 Z"
"户部灵机接话, 学户部侍郎的口气, 把'5 万石'补成'30 万石'"
"训导给的赏罚分, 灵机按簿改方向, 越改越正"
```

读者问题: "一秒懂"。

### 怎么执行
- **删除"在做 X 的时候"**: 改为"X 时"(或直接删除, 上下文已含)
- **删除"如果"**: 改为"X 的话, Y 做 Z"(或直接说"X, Y")
- **删除"就会/就会 / 就能"**: 改为"X, Y"(去掉冗余)
- **长定语前堆**: 改为"主句在前, 修饰在后"
  - ❌ "按评估结论改的灵机, 接话会变正"
  - ✅ "灵机按评估结论改, 接话变正"

### 翻译腔 → 朝代白话 速查表
| 翻译腔 | 朝代白话 |
|--------|---------|
| 在做 X 的时候 | X 时 / 直接删除 |
| 如果 X, Y 就会 Z | X, Y |
| 按 X 的 Y | 按 X, Y |
| X 的 Y 的 Z | X 的 Y, Z |
| 由于 X, 所以 Y | X, 所以 Y / 因此 Y |
| 不仅 X, 而且 Y | X, Y |
| 不光 X, 还 Y | X, Y |

## 整改 4: 对白自然(避免"臣不省 A, 省 B"长跳)

### ch11 表现
```markdown
# ❌ ch11 沈一石对白: 用"省 A 省 B"长跳, 逻辑跳
"臣的'评估派'也是为匠人好 —— 臣不省试院的考核, 省接话灵机的清静"
```

读者问题: "省 A 省 B"逻辑跳, 读者要"猜": "省 A 是为了什么? 省 B 又是为了什么?"

### ch12 整改
```markdown
# ✅ ch12 韩守拙对白: 用比喻, 不绕
"灵机是有灵性的灵物 —— 像太学的学子, 马越打越笨, 马不训反倒是好马。让灵机苦训, 
等于让学子天天挨板子, 灵性磨光了, 接话接得更差"
```

读者问题: "懂"(比喻一下就懂)。

### 怎么执行
- **对白要有"画面感"**: 用比喻(马 / 学子 / 镜子 / 衣)
- **避免"省 A 省 B" / "X 是为 Y, 不是为 Z" 句式**: 太绕
- **反派对白要"善"**: 真心关怀, 不是"我就是要反对"
- **对白长度**: 一段对白 1-3 句, 不超过 4 句
- **每段对白有 1 个"画面词"**: 马 / 学子 / 镜 / 衣 / 树 / 灯 / 河

### 比喻素材库(朝代白话常用)
- 马 / 学子 / 镜子 / 衣服(训习/评估/反派对白)
- 笔 / 墨 / 纸 / 砚(第02-06章翰林院)
- 机件 / 纸契 / 拜帖(第07-10章机件)
- 评 / 考官 / 考卷(第11章评估)
- 训导 / 学子 / 训习簿(第12章训习)

## 整改 5: M0→M5 因果链严丝合缝(无"灵机忽然懂了")

### ch11 表现
```markdown
# ❌ ch11 M3: "灵机忽然懂了"
"户部灵机按第一轮评估改进: 听评扣分 → 户部灵机纸契补全'回音'项"

# ❌ 没有具体为什么改进, 直接列改进项
```

读者问题: "为什么补'回音'项? 为什么补了'回音'就接得好了?"

### ch12 整改
```markdown
# ✅ ch12 M3: 每步都有"为什么"
"训导先做'习': 户部灵机读'户部税奏三百份', 学'户部侍郎正确的口气' —— 不补位、 
不删位、不改字, 按侍郎原本的话接话。习一周。"

"训导再做'考': 户部灵机接话后, 评估官按四评打分 —— 听评 80 / 查评 90 / 比评 95 / 判评 90。考一周。"

"训导最后'改': 户部灵机按扣分点改 —— 听评扣分(纸契缺'回音') → 补'回音'; 
查评扣分(据某档不清) → 改'据户部税奏第 7 条'; 比评扣分(差一位数) → 严格按'5 万石 15 日抵达'; 
判评扣分(事不成) → 严格按'调粮草须 5 万石 15 日抵达'。改一周。"
```

读者问题: "懂"(每步都有"为什么这样改")。

### 怎么执行
- **每一步要写"为什么"**: 不是"灵机改字", 是"灵机按扣分点改字, 因为听评扣了 10 分, 因为纸契缺'回音'"
- **每一步要写"改了什么"**: 不是"灵机改了", 是"灵机改字 / 改句 / 改口气, 改了几处"
- **每一步要写"改完得了什么分"**: "改完听评 80 → 95"
- **因果链**: "扣分 → 改 → 提分"三件套
- **避免"灵机自己懂了"**: 灵机不是自己懂, 是训导/考官一步步教

### 因果链速查模板
```markdown
[动作] → [为什么] → [改了什么] → [提分]
训导先做"习" → 让灵机学正 → 户部灵机读 300 份 → 习一周
训导再做"考" → 让灵机看清差 → 评估官按四评打分 → 听评 80
训导最后"改" → 让灵机改对方向 → 灵机按扣分点改 → 听评 95
```

## How to apply for 第13+ 章

- **整改 1**: 写完一幕先看第一句, 主旨句格式 "**主旨: X —— Y**"
- **整改 2**: M1 反派退场要明确, M5 再登场间隔 1-2 幕
- **整改 3**: 删"在做 X 的时候" / "如果 X, Y 就会 Z" / 长定语前堆
- **整改 4**: 对白用比喻(马/学子/镜/衣), 1-3 句, 1 画面词
- **整改 5**: 每步 "动作 → 为什么 → 改了什么 → 提分"

**Why**: 5 整改是把 ch11 用户反馈"软质量"问题量化为"硬规则", 跨章复用, 后续章节无 ch11 那种"故事逻辑+语序混乱"反馈。
